JavaScript Regular Expressions

Regular Expressions, known as "regex" or "regexp", are a specially formatted text strings used to find patterns in text.

JavaScript provides a list of built-in methods for performing pattern-matching.

  • exec(): Search for a match in string; returns array of information or null on mismatch.
  • test(): Test whether a string matches a pattern; returns true or false.
  • search(): Search for a match within a string; returns the index of the first match or -1 if not found.
  • replace(): Searchh for a match in a string and replaces the matched substring with a replacement string.
  • match(): Search for a match in a string; returns an array of information or null on mismatch.
  • split(): Splits up a string into an array of substrings using a regular expression.
exec() and test() accepts string as a parameter, search(), replace(), match() and split() accept regular expression as a parameter.

Defining Regular Expression

Regular expresssions, in JavaScript, are represented by RegExp Object.

Two ways of creating a new RegExp Object: using the literal syntax, RegExp() constructor

Example: var regex = /^Mr\./ or var regex = new RegExp("^Mr\\.");

Pattern Matching

Regular Expression patterns include the use of letters, digits, punctuation marks, etc., plus a set of special regular expression characters: . * ? + [ ] ( ) { } ^ $ | \

To use these characters literally, we have to backslash them.

Character Classes

Square brackets surrounding a pattern of characters are called a character class.

Examples:

  • [abc]: Matches any one of the characters a, b, or c
  • [^abc]: Matches any one character other than a, b, or c
  • [a-z]: Matches any one from lowercase a to lowercase z
  • [A-Z]: Matches any one from uppercase A to uppercase Z
  • [a-Z]: Matches any one from lowercase a to uppercase Z
  • [0-9]: Matches a single digit between 0 and 9
  • [a-z0-9]: Matches a single character between a-z or 0-9

Using test() method:

var regex = /RegEx[p]/; var str = "We are learning RegEx today. RegExp is also fun"; if(regex.test(str)) { alert("Match found!"); } else { alert("Match not found."); }

Predefined Character Classes

Some character classes such a digits, letters and whitespaces are used so frequently that there are shortcuts for them.

  • . : Matches any single character except newline \n.
  • \d : Matches any digit character. Same as [0-9]
  • \D : Matches any non-digit character. Same as [^0-9]
  • \s : Matches any whitespace character.
  • \S : Matches any non-whitespace character.
  • \w : Matches any word character. Same as [a-zA-Z_0-9]
  • \W : Matches any non-word character. Same as [^a-zA-Z_0-9]

Repetition Quantifiers

Quantifiers are used to specify how many times a character in a regular expression should match.

  • p+ : Matches one or more occurences of the letter p
  • p* : Matches zero or more occurences of the letter p
  • p? : Matches zero or one occurence of the letter p
  • p{3} : Matches exactly three occurences of the letter p
  • p{2,3} : Matches at least two occurences of letter p, but not more than three occurences
  • p{2,} : Matches two or more occurences of the letter p
  • p{,3} : Matches at most three occurences of the letter p

Position Anchors

Sometimes, we need to match at the beginning or end of a string.

Two common anchors: caret(^) representing the start of a string and the dollar ($) sign representing the end of the string.

Pattern Modifiers

A pattern modifier allows you to control the way a pattern match is handled.

Patter modifiers are placed directly after the regular expression.

  • g : Perform a global match; finds all occurences
  • i : Makes the match case-insensitive manner
  • o : Evaluates the expression only once
  • m : Changes the behavior of ^ and $ to match against a newline boundary
  • s : Changes the behavior of .(dot) to match all the chracters, including newlines

Alternation

Alternation allows to specify alternative version of a pattern. A vertical bar (|) is used to specify alternation

JavaScript Form Validation

JavaScript allows client-side form validation which is faster as the validation occurs within the user's web browser as compared to server-side validation.

Form validation process typically consists of two parts: the required fields validation and the data format validation.

JavaScript AJAX

JavaScript ES6

ES6 stands for ECMAScript 6, is the sixth and major edition of the ECMAScript language specification standard.

ES6 has introduced several new features such as template literals, new loop for iterating over arrays and objects, etc.

The let keyword

ES6 introduced the new let keyword for declaring variables. Prior to ES6, var keyword was used to declare variables.

The const keyword

ES6 have introduced the new const keyword and made it possible to define constants. Constants are read-only; cannot assign new values.

The for...of loop

The new for...of loop allows us to iterate over arrays or other iterable objects very easily.

Template Literals

Template literals provide an easy and clean way to create multi-line strings and perform string interpolation.

Template literals are creating using back-tick (``) character instead of single or double quotes.

Variables or expressions can be placed inside the ${...} syntax.

Example: let a = 10; let b = 20; let result = `The sum of ${a} and ${b} is ${a+b}`

Default values for function parameters

In ES6, we can specify default values to the function parameters.

Arrow Functions

Arrow Functions provides a concise syntax for writing function expressions by option out the function and return keywords.

Arrow Functions are defined using a new syntax, the fat array (=>) notation.

Example: var sum = (a, b) => a + b; console.log(sum(10,25));

Classes

With ES6, JavaScript have moved towards object oriented approach. Classes are introduced in ES6 that make it easier to create objects, implement inheritance using extends keyword and reuse the code.

Modules

Prior to ES6, there was no native support for modules in JavaScript.

ES6 introduced file based module, in which each module is represented by a separate .js file. Now, we can use the export or import statement ina module to export or import variables, functions, classes or any other entities.
